## 1. Brand Foundation

### 1.1 The Name: oxide-tech

The name **oxide** is a deliberate reference to **silicon dioxide (SiO₂)** — the fundamental insulator in every modern transistor. It is the literal bedrock of the computing hardware we build.

- **oxide** = The physical foundation (silicon, copper, hardware)
- **tech** = The digital intelligence (firmware, software, systems)
- **The hyphen** = The bridge between physical and digital realms

The name embodies our core identity: we are the bridge between raw physical elements and intelligent computation.

### 1.2 Brand Essence

**"The Craftsman's Guild of Embedded Systems"**

oxide-tech operates less like a modern startup and more like a medieval guild of master craftsmen. Every product is a reflection of the company's reputation. Deep mentorship, master-level work, and a long-term vision that builds for decades, not quarters.

### 1.3 Brand Pillars

| Pillar | Description |
|--------|-------------|
| **Precision** | Mathematical certainty in every line of code and copper trace |
| **Legacy** | 23 years of continuous evolution and reinvention |
| **Safety** | Functional safety as a business differentiator |
| **Mastery** | Deep expertise across the entire technology stack |
| **Independence** | First-principles thinking over imitation |

### 1.4 Brand Personality

If oxide-tech were a person, they would be:

- **The Craftsman**: Meticulous, patient, obsessed with details
- **The Guardian**: Protective, principled, uncompromising on quality
- **The Philosopher**: Thoughtful, reflective, driven by deeper meaning
- **The Teacher**: Knowledgeable, generous with expertise, mentoring
- **The Explorer**: Curious, ambitious, always reaching for the next horizon

### 1.5 Brand Archetype

**Primary Archetype**: The Creator × The Sage
**Secondary Archetype**: The Ruler × The Magician

We create with wisdom. We rule through expertise. We transform raw materials into intelligent systems.

## 3. Logo System

### 3.1 The Mark: Abstract Geometric Identity

The oxide-tech logo is an abstract geometric mark that synthesizes three core concepts:

1. **The Hexagon** (outer shape) — Represents the chemical/crystal structure of silicon dioxide
2. **The Rotated Square** (inner shape) — Represents the integrated circuit die
3. **The Central Core** (focal point) — Represents the processor core / the "oxide" at the heart

The six corner nodes represent IC pins, connecting the physical hardware to the digital intelligence within.

### 3.2 Logo Anatomy

```
┌─────────────────────────────────────────┐
│                                         │
│           ╱╲                            │
│          ╱  ╲   ← Hexagon (SiO₂)        │
│         ╱ ◇  ╲  ← Rotated Square (IC)   │
│        ╱  ●   ╲ ← Central Core          │
│         ╲     ╱                         │
│          ╲   ╱                          │
│           ╲╱                            │
│                                         │
│    ○  ○  ○  ○  ○  ○  ← IC Pins         │
│                                         │
└─────────────────────────────────────────┘
```

**Components**:
- **Outer hexagon**: Stroke `#00d4ff`, 1.5px width, 60% opacity
- **Inner rotated square**: Stroke `#00ff9f`, 1.5px width, 80% opacity
- **Circuit traces**: Stroke `#00d4ff`, 1px width, 50% opacity
- **Corner nodes**: Fill `#00d4ff`, 2px radius, 80% opacity
- **Central core**: Fill `#00d4ff`, 10px radius, 95% opacity
- **Core ring**: Fill `#07070a` (background), 5px radius
- **Core center**: Fill `#00ff9f`, 2.5px radius

### 3.3 Logo Variations

#### Primary Logo (Full Color)
- **Use**: Digital interfaces, websites, presentations
- **Background**: Dark backgrounds only (`#07070a` to `#181A20`)
- **Colors**: Cyan `#00d4ff` + Green `#00ff9f`

#### Monochrome Light
- **Use**: Dark print materials, embossing, engraving
- **Color**: White `#FFFFFF` at 90% opacity

#### Monochrome Dark
- **Use**: Light backgrounds (rare, for specific contexts)
- **Color**: Carbon `#07070a`

#### Single Color Cyan
- **Use**: Technical documentation, schematics
- **Color**: Cyan `#00d4ff` only

#### Single Color Green
- **Use**: Success states, safety contexts
- **Color**: Green `#00ff9f` only

### 3.4 Logo Lockups

#### Horizontal Lockup (Primary)
```
[MARK] oxide-tech
       ──────
       cyan   green
```

**Typography**:
- "oxide" — Space Grotesk, 600 weight, `#f8fafc`
- "-" — Space Grotesk, 600 weight, `#00ff9f`
- "tech" — Space Grotesk, 600 weight, `#00d4ff`

#### Stacked Lockup
```
   [MARK]
  oxide-tech
```

#### Mark Only
```
  [MARK]
```
- **Use**: Favicons, app icons, social media avatars

### 3.5 Clear Space

The logo must always be surrounded by clear space equal to the height of the central core element (the small cyan circle).

```
┌─────────────────────────────┐
│                             │
│      ← clear space →        │
│        ┌───────┐            │
│        │ [LOGO] │           │
│        └───────┘            │
│      ← clear space →        │
│                             │
└─────────────────────────────┘
```

**Minimum clear space**: 25% of logo width on all sides

### 3.6 Minimum Sizes

| Context | Minimum Size |
|---------|--------------|
| Digital (screen) | 32px height |
| Print (offset) | 12mm height |
| Favicon | 16×16px (simplified mark) |
| Social media avatar | 48×48px |

Below these sizes, use the simplified mark (hexagon + core only, no inner details).

### 3.7 Logo Misuse — NEVER Do This

- ❌ Do not rotate the logo (except the 180° hover animation on websites)
- ❌ Do not change the colors outside the approved palette
- ❌ Do not add drop shadows, glows, or effects
- ❌ Do not stretch or distort the proportions
- ❌ Do not place on busy backgrounds without proper contrast
- ❌ Do not recreate the logo with different shapes
- ❌ Do not use gradients on the mark itself
- ❌ Do not separate the mark from the wordmark in horizontal lockup
- ❌ Do not use the logo at sizes below the minimum
- ❌ Do not place the logo on low-contrast backgrounds

## 5. Typography System

### 5.1 Typography Philosophy

oxide-tech typography balances three qualities:
1. **Technical precision** — Monospaced fonts for data and code
2. **Modern elegance** — Geometric sans-serif for headlines
3. **Human readability** — Optimized sans-serif for body text

### 5.2 Font Families

#### Primary Typeface: Space Grotesk
- **Use**: Headlines, display text, logo
- **Weights**: 300, 400, 500, 600, 700
- **Characteristics**: Geometric, technical, slight industrial flair
- **Fallback**: `system-ui, -apple-system, sans-serif`

#### Monospace Typeface: JetBrains Mono
- **Use**: Code, data, labels, technical content, UI chrome
- **Weights**: 400, 500, 600, 700
- **Characteristics**: Highly legible, technical, developer-friendly
- **Fallback**: `ui-monospace, SFMono-Regular, monospace`

#### Body Typeface: Inter
- **Use**: Body text, descriptions, long-form content
- **Weights**: 300, 400, 500, 600, 700
- **Characteristics**: Clean, highly legible, professional
- **Fallback**: `system-ui, -apple-system, sans-serif`

### 5.3 Type Scale

Based on a 1.25 (Major Third) scale with 16px base:

| Level | Size | Line Height | Weight | Letter Spacing | Font | Usage |
|-------|------|-------------|--------|----------------|------|-------|
| **Display** | 4.5rem (72px) | 1.05 | 700 | -0.03em | Space Grotesk | Hero headlines |
| **H1** | 3.5rem (56px) | 1.1 | 700 | -0.02em | Space Grotesk | Page titles |
| **H2** | 2.5rem (40px) | 1.15 | 700 | -0.02em | Space Grotesk | Section titles |
| **H3** | 2rem (32px) | 1.2 | 600 | -0.01em | Space Grotesk | Subsections |
| **H4** | 1.5rem (24px) | 1.3 | 600 | -0.01em | Space Grotesk | Card titles |
| **H5** | 1.25rem (20px) | 1.4 | 600 | 0 | Space Grotesk | Small titles |
| **H6** | 1.1rem (18px) | 1.4 | 600 | 0 | Space Grotesk | Minor titles |
| **Body Large** | 1.1rem (18px) | 1.7 | 400 | 0 | Inter | Lead paragraphs |
| **Body** | 1rem (16px) | 1.6 | 400 | 0 | Inter | Standard body |
| **Body Small** | 0.9rem (14px) | 1.6 | 400 | 0 | Inter | Secondary text |
| **Caption** | 0.8rem (13px) | 1.5 | 400 | 0 | Inter | Captions |
| **Label** | 0.75rem (12px) | 1.4 | 500 | 0.1em | JetBrains Mono | Labels |
| **Code** | 0.85rem (14px) | 1.6 | 400 | 0 | JetBrains Mono | Code blocks |
| **Data** | 0.7rem (11px) | 1.4 | 500 | 0.15em | JetBrains Mono | Data, stats |

### 5.4 Typography Rules

#### Headlines
- Use **Space Grotesk** for all headlines
- Negative letter-spacing (-0.02em to -0.03em) for large sizes
- Gradient text allowed for hero headlines only
- Never use all-caps for headlines longer than 3 words

#### Body Text
- Use **Inter** for body text
- Maximum line length: 65-75 characters
- Line height: 1.6-1.7 for readability
- Paragraph spacing: 1.5em

#### Technical Content
- Use **JetBrains Mono** for:
  - Code snippets
  - Data values
  - Labels and tags
  - Navigation items
  - Timestamps
  - Technical specifications
- Uppercase with letter-spacing (0.1-0.2em) for labels
- Never use for long-form body text

### 5.5 Special Typography Treatments

#### Gradient Text
```css
background: linear-gradient(135deg, #00d4ff, #00ff9f);
-webkit-background-clip: text;
-webkit-text-fill-color: transparent;
```
**Usage**: Hero headlines, key stats, special callouts

#### Monospace Labels
```css
font-family: 'JetBrains Mono', monospace;
font-size: 0.7rem;
text-transform: uppercase;
letter-spacing: 0.15em;
color: #00d4ff;
```
**Usage**: Section labels, tags, metadata

#### Blinking Cursor
```css
width: 8px;
height: 1.1em;
background: #00d4ff;
animation: blink 1s step-end infinite;
```
**Usage**: Hero headlines, terminal-style content

## 7. Imagery & Photography

### 7.1 Photography Style

oxide-tech imagery should convey:
- **Precision**: Sharp focus, clean compositions
- **Technology**: Embedded systems, PCBs, lab equipment
- **Craftsmanship**: Close-ups of hardware, hands at work
- **Scale**: Macro shots of components, wide shots of systems

### 7.2 Image Treatment

All imagery should be treated with:
- **Dark overlay**: 20-40% dark gradient for text legibility
- **Cyan tint**: Subtle cyan color grading (5-10%)
- **High contrast**: Bold differences between light and dark
- **Sharp focus**: No soft focus or bokeh (except artistic contexts)

### 7.3 Image Sources

**Primary**: Custom photography of oxide-tech products and processes
**Secondary**: AI-generated imagery for conceptual visualization
**Tertiary**: Stock photography (only when custom is not possible)

### 7.4 Image Specifications

#### Product Images
- Resolution: 2400×1600px minimum
- Format: PNG (with transparency) or high-quality JPEG
- Background: Dark gradient or transparent
- Lighting: Dramatic, side-lit, highlighting textures

#### Hero Images
- Resolution: 3840×2160px (4K)
- Aspect ratio: 16:9 or 21:9
- Treatment: Dark overlay for text legibility

#### Card Images
- Resolution: 1200×800px
- Aspect ratio: 3:2
- Treatment: Gradient fade at bottom for text overlay

### 7.5 Prohibited Imagery

- ❌ Generic stock photos of people in suits
- ❌ Overly processed or filtered images
- ❌ Bright, colorful, "startup-y" imagery
- ❌ Images that don't relate to embedded systems
- ❌ Low-resolution or pixelated images
- ❌ Images with visible watermarks

## 8. Graphic Elements

### 8.1 PCB Background Pattern

The signature oxide-tech background is an animated PCB trace pattern that evokes the physical nature of our work.

**Implementation**:
- Canvas-based animation
- Random trace generation with 90° angles
- Animated signal flow (progressive drawing)
- Via holes at connection points
- Dual-color: Cyan and Green traces
- Low opacity (0.35) for subtlety

**Technical Specs**:
```javascript
- Trace density: 1 per 45,000px²
- Segments per trace: 3-8
- Segment length: 20-100px
- Animation speed: 0.0005-0.002 progress/frame
- Colors: #00d4ff (50%), #00ff9f (50%)
- Line width: 0.5-2px
- Opacity: 0.1-0.4
```

### 8.2 Scanline Overlay

A subtle CRT-inspired scanline effect that adds texture and depth.

**Implementation**:
```css
background: repeating-linear-gradient(
  0deg,
  transparent,
  transparent 2px,
  rgba(0, 212, 255, 0.008) 2px,
  rgba(0, 212, 255, 0.008) 4px
);
```

**Usage**: Fixed overlay across entire viewport, very subtle

### 8.3 Grid Overlay

A technical grid that reinforces the precision aesthetic.

**Implementation**:
```css
background-image: 
  linear-gradient(rgba(0, 212, 255, 0.02) 1px, transparent 1px),
  linear-gradient(90deg, rgba(0, 212, 255, 0.02) 1px, transparent 1px);
background-size: 60px 60px;
```

**Usage**: Fixed overlay, subtle technical feel

### 8.4 Noise Texture

A subtle noise overlay that adds film-like grain and prevents banding.

**Implementation**:
```css
opacity: 0.03;
background-image: url("data:image/svg+xml,..."); /* SVG noise filter */
```

**Usage**: Fixed overlay, very subtle texture

### 8.5 Hologram Effect

The signature hologram viewer component for showcasing products.

**Components**:
- **Floating animation**: 8s ease-in-out infinite
- **Glow effect**: Drop-shadow with hue-rotate
- **Orbital rings**: 3 concentric rings at different speeds
- **Scanline**: Horizontal scan moving top to bottom
- **Data overlays**: Technical readouts at corners

**Technical Specs**:
```css
- Float range: 0 to -20px Y
- Rotation: ±5° Y, ±2° X
- Glow: 30-50px drop-shadow
- Ring 1: 340px, 10s rotation
- Ring 2: 390px, 14s reverse rotation
- Ring 3: 440px, 20s rotation (dashed)
- Scanline: 3.5s linear infinite
```

### 8.6 Gradient Borders

Cards and containers use subtle gradient borders on hover.

**Implementation**:
```css
::before {
  content: '';
  position: absolute;
  top: 0; left: 0; right: 0;
  height: 2px;
  background: linear-gradient(90deg, #00d4ff, #00ff9f);
  opacity: 0;
  transition: opacity 0.4s ease;
}

:hover::before {
  opacity: 1;
}
```

### 8.7 Glow Effects

Interactive elements emit subtle glows on hover.

**Implementation**:
```css
box-shadow: 0 0 30px rgba(0, 212, 255, 0.1);
```

**Usage**: Cards, buttons, interactive elements

## 9. Motion & Animation

### 9.1 Motion Philosophy

oxide-tech motion design should feel:
- **Purposeful**: Every animation serves a function
- **Smooth**: Cubic-bezier easing, never linear (except infinite loops)
- **Subtle**: Enhance, don't distract
- **Technical**: Precise timing, engineering-inspired

### 9.2 Easing Functions

| Name | Value | Usage |
|------|-------|-------|
| **Smooth** | `cubic-bezier(0.16, 1, 0.3, 1)` | General transitions |
| **Bounce** | `cubic-bezier(0.68, -0.55, 0.265, 1.55)` | Playful interactions |
| **Sharp** | `cubic-bezier(0.4, 0, 0.2, 1)` | Quick UI transitions |
| **Linear** | `linear` | Infinite loops only |

### 9.3 Duration Guidelines

| Type | Duration | Usage |
|------|----------|-------|
| **Micro** | 150-200ms | Button hovers, color changes |
| **Small** | 300-400ms | Card hovers, menu transitions |
| **Medium** | 500-800ms | Section reveals, page transitions |
| **Large** | 1000-1500ms | Hero animations, complex sequences |
| **Infinite** | 3s-20s | Background animations, loops |

### 9.4 Reveal Animations

#### Fade Up
```css
opacity: 0;
transform: translateY(40px);
transition: opacity 1s, transform 1s;

.visible {
  opacity: 1;
  transform: translateY(0);
}
```

#### Stagger Reveal
```css
:nth-child(1) { transition-delay: 0s; }
:nth-child(2) { transition-delay: 0.1s; }
:nth-child(3) { transition-delay: 0.2s; }
/* ... up to 8 children */
```

#### Line Reveal (Hero)
```css
transform: translateY(110%);
animation: lineReveal 1s cubic-bezier(0.16, 1, 0.3, 1) forwards;
```

### 9.5 Interactive Animations

#### Button Hover
```css
transform: translateY(-3px);
box-shadow: 0 12px 40px rgba(0, 212, 255, 0.4);
```

#### Card Hover
```css
transform: translateY(-6px);
border-color: rgba(0, 212, 255, 0.3);
box-shadow: 0 30px 80px rgba(0, 212, 255, 0.12);
```

#### Magnetic Effect
```javascript
// Buttons follow cursor slightly
transform: translate(${x * 0.15}px, ${y * 0.15}px);
```

### 9.6 Loading States

#### Page Loader
- Logo pulse animation (1.5s infinite)
- Progress bar fill (2s ease-out)
- Fade out on complete (0.8s)

#### Skeleton Screens
- Shimmer effect with gradient
- 1.5s duration, infinite loop
- Subtle, not distracting

### 9.7 Reduced Motion

Respect user preferences:
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 13. Do's and Don'ts

### 13.1 Logo Do's

- ✅ Use approved color variations only
- ✅ Maintain clear space around logo
- ✅ Use at or above minimum sizes
- ✅ Place on approved backgrounds
- ✅ Use the horizontal lockup as primary
- ✅ Align logo to grid when possible

### 13.2 Logo Don'ts

- ❌ Rotate or skew the logo
- ❌ Change colors outside approved palette
- ❌ Add effects (shadows, glows, bevels)
- ❌ Stretch or distort proportions
- ❌ Place on busy/low-contrast backgrounds
- ❌ Recreate with different shapes
- ❌ Use below minimum sizes
- ❌ Separate mark from wordmark (in lockup)

### 13.3 Color Do's

- ✅ Use the full palette as specified
- ✅ Maintain contrast ratios for accessibility
- ✅ Use gradients only as specified
- ✅ Test colors on multiple devices
- ✅ Use semantic colors correctly

### 13.4 Color Don'ts

- ❌ Use unapproved colors
- ❌ Create new gradients
- ❌ Use low-contrast combinations for text
- ❌ Use bright colors for large areas
- ❌ Mix color systems (RGB/CMYK) incorrectly

### 13.5 Typography Do's

- ✅ Use approved typefaces only
- ✅ Follow the type scale
- ✅ Maintain proper line heights
- ✅ Use appropriate weights for hierarchy
- ✅ Test readability at all sizes

### 13.6 Typography Don'ts

- ❌ Use unapproved typefaces
- ❌ Mix too many typefaces
- ❌ Use decorative fonts
- ❌ Ignore line length guidelines
- ❌ Use all-caps for long text

### 13.7 Imagery Do's

- ✅ Use high-resolution images
- ✅ Apply brand treatments
- ✅ Focus on technical/precision themes
- ✅ Ensure proper licensing
- ✅ Optimize for web

### 13.8 Imagery Don'ts

- ❌ Use low-resolution images
- ❌ Use generic stock photos
- ❌ Apply heavy filters
- ❌ Use bright/colorful imagery
- ❌ Include watermarks

### 13.9 Motion Do's

- ✅ Use purposeful animations
- ✅ Follow duration guidelines
- ✅ Use approved easing functions
- ✅ Respect reduced motion preferences
- ✅ Test performance

### 13.10 Motion Don'ts

- ❌ Animate without purpose
- ✅ Use linear easing (except loops)
- ❌ Create distracting animations
- ❌ Ignore accessibility
- ❌ Over-animate
